# README.md と CLAUDE.md の整理計画

## 目的

README.md から開発者向け情報を CLAUDE.md に移動し、README.md を利用者向けに簡潔化する。

## 現状の問題

README.md（567行）に以下が混在：
- 利用者向け情報（セットアップ、使い方、機能説明、トラブルシューティング）
- 開発者向け情報（アーキテクチャ、ファイル構造、開発コマンド、テスト、依存関係）

## プランファイルのリネーム

完了後、このファイルを `./plans/2026-01-27-readme-claude-md-reorganization.md` にリネーム

## 変更方針

### README.md（利用者向け）

**残す内容：**
- 概要・対象ソース
- セットアップ手順（Deno, GitHub Discussions, Claude Code Action, シークレット, ミュートワード）
- 使い方（自動実行・手動実行）
- 機能説明（ラベル付与、メンション、@claude、ミュートワード）
- トラブルシューティング
- ライセンス

**削除する内容：**
- アーキテクチャ図・設計ポイント
- ファイル構造
- ローカルテスト（セットアップ内）
- JSONデータフォーマット
- カスタマイズセクション
- 開発セクション全体（プレビュー、テスト、品質チェック、依存関係）
- 週次オーケストレーター（機能セクション内）
- URL正規化機能（利用者が意識する必要がない）

### CLAUDE.md（開発者向け）

**追加する内容：**
- ファイル構造（詳細版）
- JSONデータフォーマット
- カスタマイズ方法
- プレビューの詳細
- 依存関係
- 要約JSONフォーマット

## 対象ファイル

- `README.md`
- `CLAUDE.md`

## 検証方法

- README.md が利用者視点で読みやすいか確認
- CLAUDE.md に開発に必要な情報が揃っているか確認
- 重複がないか確認
